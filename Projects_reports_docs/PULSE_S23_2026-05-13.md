### Le problème

`ToolStatsStore` existe, `blended_effectiveness()` existe,
mais personne n'écrit dedans après exécution.
L'IDE donne toujours nuclei à 0.95 — score V6 figé.
Le LLM se nourrit de données statiques, pas terrain.

### Le fix — `mcp_core/server_setup.py` dans `finalize()`

Cherche la fonction `finalize()` interne à `run_security_tool()`.
Elle fait déjà `_op_metrics.record(_telemetry)`.
Ajoute juste après :

```python
# Ferme la boucle IDE — nourrit blended_effectiveness()
try:
    get_tool_stats_store().record(
        tool_name,
        success=bool(_telemetry.get("success", False))
    )
except Exception as exc:  # jamais bloquer l'exécution principale
    logger.debug("tool_stats_store.record failed: %s", exc)
```

**Vérifie d'abord comment `_tool_stats` est importé dans `server_setup.py` :**

```bash
grep -n "tool_stats\|ToolStatsStore" mcp_core/server_setup.py | head -10
```

Si c'est un singleton lazy, utilise le getter existant.
Si c'est importé directement, adapte en conséquence.
