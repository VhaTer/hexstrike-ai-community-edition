#### Panel 3 — Performance (durées par tool)

Source : `_op_metrics.summary()` — champ `avg_duration` par tool.

```python
@app.tool("get_performance_stats")
async def get_performance_stats() -> dict:
    summary = get_op_metrics().summary()
    return {
        tool: {
            "avg_duration": stats.get("avg_duration", 0),
            "runs": stats.get("runs", 0),
            "timed_out": stats.get("timed_out", 0)
        }
        for tool, stats in summary.get("tools", {}).items()
        if stats.get("runs", 0) > 0
    }
```

UI : bar chart horizontal par tool, durée moyenne, badge "slow" si > 60s.

#### Panel 4 — Cache

Source : `health://server` → `cache_stats`.

```python
@app.tool("get_cache_stats")
async def get_cache_stats() -> dict:
    health = get_server_health()
    cache = health.get("cache_stats", {})
    hits = cache.get("hits", 0)
    misses = cache.get("misses", 0)
    total = hits + misses
    return {
        "hits": hits,
        "misses": misses,
        "hit_rate": round(hits / total * 100, 1) if total > 0 else 0,
        "entries": cache.get("size", 0),
        "time_saved_seconds": hits * cache.get("avg_duration_saved", 0)
    }
```

UI : ring chart hit/miss + légende hits/misses/entrées/temps économisé.

#### Panel 5 — Sécurité terrain (destructive tools)

Source : `_op_metrics.summary()` — champ `confirmations` par tool.

```python
@app.tool("get_security_events")
async def get_security_events() -> dict:
    summary = get_op_metrics().summary()
    destructive = ["metasploit", "aireplay_ng", "responder",
                   "mitm6", "sqlmap", "hydra", "hashcat"]
    return {
        tool: summary.get("tools", {}).get(tool, {}).get("confirmations", {})
        for tool in destructive
        if tool in summary.get("tools", {})
    }
```

UI : table — tool / action / statut (accepté / refusé / en attente / passif).

#### Panel 6 — Rate limits

Source : `_op_metrics.summary()` — champ `rate_limit_detected` + `opt_profile`.

```python
@app.tool("get_rate_limit_stats")
async def get_rate_limit_stats() -> dict:
    summary = get_op_metrics().summary()
    return {
        tool: {
            "target": stats.get("last_target"),
            "profile_forced": stats.get("opt_profile"),
            "confidence": stats.get("rate_limit_confidence", 0)
        }
        for tool, stats in summary.get("tools", {}).items()
        if stats.get("rate_limit_detected", False)
    }
```

UI : liste cibles détectées + profil forcé automatiquement (stealth/normal) + confidence %.

#### Panel 7 — Intelligence (scores live vs baseline)

Source : `intelligence://tools` — ToolStatsStore feed fermé en S26.

```python
@app.tool("get_intelligence_stats")
async def get_intelligence_stats() -> dict:
    store = get_tool_stats_store()
    all_stats = store.get_all_stats()
    result = {}
    for tool, stats in all_stats.items():
        runs = stats.get("runs", 0)
        if runs == 0:
            continue
        baseline = get_tool_baseline(tool)  # depuis tool_registry
        live = store.live_effectiveness(tool)
        blended = store.blended_effectiveness(tool, baseline)
        result[tool] = {
            "runs": runs,
            "baseline": round(baseline, 2),
            "live": round(live, 2) if live is not None else None,
            "blended": round(blended, 2),
            "drift": round(blended - baseline, 2) if live is not None else 0,
            "adapted": live is not None  # True si >= 5 runs
        }
    return result
```

UI : table tool / baseline / live / drift (rouge si négatif, vert si positif) / badge "adaptatif" ou "blind".

Ce panel rend visible que le LLM se nourrit de vraies données terrain.
