## 3. Architecture

```
FastMCP 3.2.4 (mcp.tool timeout=None)
  │
  ├── setup_mcp_server_standalone()  [mcp_core/server_setup.py — 1609 lignes]
  │   │
  │   ├── DIRECT_TOOLS dict (131 entrées)
  │   │   └── route vers 16 modules *_direct.py
  │   │       ├── recon_direct.py        ─── subfinder, amass, whois, dnsrecon, ...
  │   │       ├── net_scan_direct.py     ─── nmap, masscan, rustscan
  │   │       ├── web_scan_direct.py     ─── sqlmap, xsser, commix, dirsearch
  │   │       ├── web_recon_direct.py    ─── whatweb, wappalyzer, wafw00f
  │   │       ├── web_fuzz_direct.py     ─── ffuf, gobuster, feroxbuster
  │   │       ├── web_probe_direct.py    ─── httpx, nikto, joomscan, droopescan
  │   │       ├── misc_direct.py         ─── nuclei, searchsploit, libc, hashid
  │   │       ├── vuln_intel_direct.py   ─── vulnx, testssl
  │   │       ├── exploit_framework_direct.py ─── metasploit, empire, starkiller
  │   │       ├── wifi_direct.py         ─── aircrack-ng suite, bettercap
  │   │       ├── password_cracking_direct.py ─── john, hashcat, hydra
  │   │       ├── smb_enum_direct.py     ─── crackmapexec, smbclient, impacket
  │   │       ├── osint_direct.py        ─── sherlock, theHarvester, holehe
  │   │       ├── security_direct.py     ─── defender, checksec, lynis
  │   │       ├── ad_direct.py           ─── bloodhound, ldapdomaindump, mitm6
  │   │       └── ctf_direct.py          ─── CTF-specific tools
  │   │
  │   ├── run_security_tool() ─── exécuteur central
  │   │   ├── EnhancedCommandExecutor(timeout=COMMAND_TIMEOUT)
  │   │   │   └── subprocess.run() + terminate/kill
  │   │   ├── _ScanCache (adaptatif 30-90min TTL, max 500)
  │   │   ├── _op_metrics.record() (per-tool stats)
  │   │   └── polling loop ctx.report_progress() tous les 12-15s
  │   │
  │   ├── 4 MCP Resources
  │   │   ├── health://server       → status, uptime, tools_count, cache_stats, op_metrics
  │   │   ├── scan://{target}/{tool} → résultat détaillé d'un scan
  │   │   ├── metrics://tools       → success_rate, error_count, slowest_tools
  │   │   └── errors://statistics   → total_errors, types, recent
  │   │
  │   └── MCP Tools
  │       ├── server_health()       → wrapper autour de health://server
  │       ├── plan_attack()         → IDE intelligence / CTF dispatch
  │       ├── validate_environment()→ 114 binaires, version detection
  │       └── get_tool_skill()      → tool-to-skill mapping
  │
  └── HTTP Custom Routes [hexstrike_server.py]
      ├── GET /dashboard            → server_static/index.html
      ├── GET /web-dashboard        → JSON complet (health + op_metrics + errors + scans)
      ├── GET /web-dashboard/stream → SSE (push toutes les 2s)
      ├── GET /api/logs/stream      → SSE logs temps réel (buffer 1000 lignes)
      ├── GET /health               → readiness probe (200/503/500)
      └── GET /ping                 → liveness probe
```

### Hiérarchie des Timeouts

```
FastMCP mcp.tool(timeout=None)                    ← aucun timeout framework
  └→ run_security_tool() polling loop             ← ctx.report_progress toutes les 12-15s
       └→ EnhancedCommandExecutor(timeout=300)    ← vrai timeout + terminate/kill
```

### Cache

- `AdvancedCache` (LRU + TTL) → `_ScanCache` (adaptatif)
- Clé = `{session_id}:{tool}:{target}:[hash(params)]`
- TTL adaptatif : 30 min (exec < 10s) / 60 min (10-60s) / 90 min (>60s)
- Max 500 entrées, LRU eviction, thread daemon cleanup toutes les 60s

## 5. Dashboard

**URL** : `http://host:8888/dashboard`

**Stack** : HTML/JS pur — zéro build, zéro npm, zéro node_modules. Servi par Starlette StaticFiles.

### Onglet Dashboard (6 panneaux)

| Panneau | Données | Source SSE |
|---------|---------|------------|
| Status Bar | status, version, uptime, commands, success rate, tools, cache, errors | `/web-dashboard/stream` (2s) |
| Resource Gauges | CPU (cramoisi), Mem (ambre), Disk (rouge), ↑↓ réseau, RAM/disque libre | `/web-dashboard/stream` |
| Execution Activity | 20 derniers scans, triés par timestamp, pulse < 5s | `/web-dashboard/stream` |
| Sparkline + Slowest | CPU/Mem 30pts canvas + top 5 tools by avg duration | `/web-dashboard/stream` |
| Cache + Errors | hits/misses/evictions + errors by type, recent 3 | `/web-dashboard/stream` |
| Categories | Grille pliable, disponible/total par catégorie | `/web-dashboard/stream` |

### Onglet Logs

- SSE temps réel via `/api/logs/stream?lines=150`
- Buffer 1000 lignes, affichage 50/100/200/500
- Filtres : search text, niveau (ALL/ERROR/WARN/INFO/DEBUG)
- Contrôles : pause/resume, auto-scroll, word wrap, clear
- Keepalive toutes les 5s

### Onglet Help

- Configs IDE : Claude Desktop, VS Code, OpenCode, Roo Code
- Tableau des flags MCP CLI
- Section authentification (Bearer token)
- Demo mode

### Thème

- Couleurs V6 : `--accent:#dc143c` (cramoisi), `--dark-red:#8b0000`, `--rose:#e63946`
- Logo : hexagone + ligne de pulsation en SVG (topbar + favicon)
